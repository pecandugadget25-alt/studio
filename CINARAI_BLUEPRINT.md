# CINARAI BLUEPRINT

Subtitle

Official Product Blueprint

Version 1.0

## 1. Project Vision

CINARAI is not a conventional LMS. It is a guided learning experience designed for focused study and sequential progress. Students do not explore random features or browse disconnected tools. Students follow one structured learning journey from beginning to end. Learning must always be simple, sequential, and easy to understand. Every experience must reduce cognitive load and keep attention on the learning task.

## 2. Core Philosophy

The following principles define the product:

- Simplicity over complexity.
- Reading before interaction.
- One journey, one direction.
- Learning first, technology second.
- Students never get lost.
- Every screen must have one clear purpose.
- Every click must move learning forward.
- Remove anything that distracts from learning.

## 3. Official Learning Flow

The learning flow must always follow this sequence:

```text
Login
↓
Dashboard
↓
Continue Learning
↓
Comic Reader
↓
Cover
↓
Contextualization
↓
Identification
↓
Navigation
↓
Argumentation
↓
Resolution
↓
Application
↓
Introspection
↓
Learning Result
↓
Return Dashboard
```

This flow must NEVER be changed.

## 4. Dashboard Blueprint

The Dashboard is intentionally minimal. It contains only the following:

- Greeting
- Current Comic
- Learning Progress
- Continue Learning button
- Locked Comics

Nothing else.

The Dashboard must NOT contain:

- Hero banner
- Marketing content
- Large illustrations
- Learning Journey cards
- CINARAI stages
- Location cards
- Statistics
- Decorative widgets
- Large information panels
- Marketplace layout

## 5. Comic Library Blueprint

The comic library is not a marketplace. It is a simple progression space for reading.

It only contains:

- Continue Reading
- Locked Comics
- Completed Comics
- Progress

The experience must not feel like a catalog or a shopping interface.

## 6. Comic Reader Blueprint

Reading is the core experience of the application.

The reader must:

- Occupy most of the screen
- Use a minimal header
- Use minimal navigation
- Provide a large reading area
- Save progress automatically
- Remain responsive across all supported devices

The reader should feel like reading a real comic, not interacting with a document workspace.

## 7. CINARAI Learning Stages

The following stages define the learning experience:

| Stage | Purpose |
| --- | --- |
| Cover | Introduces the learning context and prepares the student for the journey. |
| Contextualization | Helps the student understand the situation before deeper analysis. |
| Identification | Helps the student recognize key elements of the learning task. |
| Navigation | Guides the student through the learning path with clarity and direction. |
| Argumentation | Encourages the student to build and justify understanding. |
| Resolution | Supports the student in reaching a conclusion. |
| Application | Helps the student apply what has been learned. |
| Introspection | Encourages reflection on the learning process. |
| Learning Result | Presents the outcome of the completed journey. |

Students never see all stages on the Dashboard. Stages appear only when students reach them.

## 8. Navigation Rules

The following navigation rules apply:

- Dashboard
- Comic
- Scan AR
- Profile
- Leaderboard

Leaderboard must not become a primary navigation item. It belongs inside Profile or Achievement.

## 9. Responsive Design Rules

The experience must be designed Mobile First and then adapted for larger screens.

Supported widths include:

- 360
- 390
- 414
- 768
- 1024
- 1440

The layout must avoid:

- Horizontal scrolling
- Oversized cards
- Empty white areas
- Unnecessary spacing

## 10. UI Design Principles

The visual language must be:

- Minimal
- Calm
- Educational
- Focused
- Readable
- Book-like

The experience should feel inspired by:

- Apple Books
- Kindle
- Duolingo simplicity

It must avoid:

- Marketplace UI
- Admin Dashboard UI
- Analytics Dashboard
- Corporate SaaS UI

## 11. Things AI Agents Must Never Do

AI agents must not:

- Redesign the learning flow
- Rename CINARAI stages
- Merge stages
- Split stages
- Invent new navigation
- Invent dashboards
- Create unnecessary cards
- Create decorative sections
- Change Firestore schema
- Change authentication
- Change AI logic
- Change AR logic
- Change QR logic

## 12. AI Agent Working Rules

Every AI agent must:

- Read this document before editing
- Audit the current implementation
- Compare the implementation with the blueprint
- List mismatches
- Explain planned modifications
- Modify only requested files
- Never improvise

Blueprint has absolute priority over AI suggestions.

## 13. Success Criteria

The application is considered complete only if:

- Students immediately know what to do
- Students focus on learning instead of navigating
- Dashboard fits within one mobile screen
- Comic reader dominates the learning experience
- Every CINARAI stage appears only at the correct time
- The entire application feels like one continuous learning journey

## 14. Final Principle

This blueprint is the single source of truth for the CINARAI project.

If any AI suggestion conflicts with this blueprint, the blueprint always wins.
